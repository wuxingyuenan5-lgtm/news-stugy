# 实施路线图 V0.2

## Phase 0：确定报告成品

目标：先明确最终每天和每周要读什么，再决定收集什么信息。

交付物：

- 日报模板
- 周报模板
- 单条信息写作格式
- 报告长度与数量约束
- 引用和审核规则

验收标准：能够仅靠人工资料完成一份结构清晰、可在10至15分钟内读完的日报。

## Phase 1：建立核心信息源库

目标：建立一套规模有限、质量可控的信息源清单。

工作内容：

- 按宏观、美股、A股、贵金属、加密、能源、AI科技分类；
- 区分官方来源、专业媒体、机构研究、Newsletter和社交媒体；
- 记录获取方式、语言、更新频率和使用目的；
- 明确哪些来源每天看、每周看或仅按需查看；
- 删除重复、低质量和长期无有效信息的来源。

首批建议控制在20至40个核心来源，而不是追求全面覆盖。

验收标准：每个市场都有少量稳定的一手来源和补充来源，并且能够说明每个来源为什么值得保留。

## Phase 2：人工试运行

目标：不用复杂代码，连续跑通日报和周报。

工作流程：

1. 将候选链接放入每日收集箱；
2. 使用AI进行摘要、翻译和基础分类；
3. 人工标记高、中、低重要性；
4. 选择内容进入日报；
5. 周末从本周日报和收集箱中整理周报；
6. 记录耗时、遗漏、重复和无价值栏目。

试运行周期：至少两个完整交易周。

验收标准：

- 日报能够稳定生成；
- 每条重要内容可回溯来源；
- 人工整理时间逐步下降；
- 能够识别真正需要自动化的步骤；
- 模板经过实际阅读后完成至少一次修改。

## Phase 3：轻量自动化

目标：只自动化已经被证明重复且耗时的步骤。

优先顺序：

1. RSS或API获取固定来源；
2. 自动保存标题、时间、链接和正文摘要；
3. 基础市场分类；
4. 明显重复内容识别；
5. 按模板生成日报草稿；
6. 自动归档历史报告。

可采用Python脚本、GitHub Actions、Markdown和JSON或CSV完成。此阶段仍不要求数据库和网页前端。

验收标准：自动化减少机械操作，但不会让人工审核队列明显膨胀。

## Phase 4：内容体系优化

目标：根据真实使用体验逐步提高报告质量。

可选择增加：

- 重点主题持续跟踪；
- 经济数据前值、预期值和实际值；
- 事件更新状态；
- 市场数据和图表；
- 简要影响分析；
- 个性化关注权重；
- 全文搜索。

每次只增加一至两个明确有用的能力，并观察其是否改善阅读和研究效率。

## Phase 5：研究增强

只有在前述流程稳定后，再评估：

- 复杂事件聚类；
- 跨资产影响机制；
- 观点管理；
- 预测验证；
- 数据库和研究工作台；
- 向量检索；
- 多智能体协作。

这些属于长期优化方向，不是当前项目成功的前提。

## 当前优先级

1. 定稿日报和周报格式；
2. 建立首批核心信息源清单；
3. 定义每日收集箱格式；
4. 连续人工运行两周；
5. 根据实际问题修改来源和模板；
6. 再决定第一批自动化功能。
