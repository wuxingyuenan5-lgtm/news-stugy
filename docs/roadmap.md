# 实施路线图

## Phase 0：研究规则定稿

目标：先定义系统如何判断信息，而不是先写爬虫。

交付物：

- 研究分类与标签体系
- 信息源分级规则
- 事件卡片字段
- 重要性评分规则
- 日报与周报模板
- 人工审核标准

验收标准：同一条信息由不同时间录入时，分类、评分和处理方式基本一致。

## Phase 1：人工 MVP

目标：用最少代码跑通完整闭环。

功能：

- 通过命令行或表单手工录入链接和文本
- 保存原始文档
- 调用模型生成事件卡片草稿
- 人工修改并确认
- 生成 Markdown 日报
- 保存最终报告与引用

建议只接入 10–20 个高价值来源，不追求覆盖面。

验收标准：连续 10 个交易日生成日报，单日报人工处理时间控制在 30–60 分钟。

## Phase 2：稳定采集与去重

目标：减少重复阅读和机械操作。

功能：

- RSS、API 和固定网页采集
- 抓取日志与失败重试
- 文本哈希去重
- 同事件聚类
- 新增、更新、纠正状态识别
- 来源健康监控

验收标准：重复信息不再反复进入人工审核队列，核心来源漏抓可被发现。

## Phase 3：研究工作台

目标：将事件、观点和报告连接起来。

功能：

- 待审核事件队列
- 事件原文与结构化字段对照
- 跨资产传导链编辑
- 当前观点与失效条件
- 观察清单
- 历史报告搜索
- 日报、周报一键生成

技术建议：FastAPI + 轻量前端；也可以先用 Streamlit 验证交互。

## Phase 4：市场数据与量化指标

目标：让事件分析与价格、资金流和仓位数据联动。

功能：

- 跨资产行情快照
- 收益率、波动率和相关性
- 利率曲线与实际利率
- ETF 流量
- 期货持仓、基差、资金费率和 OI
- 市场宽度与风险偏好指标
- 事件前后资产反应

验收标准：日报中的市场描述由结构化数据生成，并可追溯计算口径。

## Phase 5：观点验证与研究评估

目标：判断系统是否真的改善研究，而不是只提高信息产量。

功能：

- 观点版本管理
- 预期方向与实际结果对照
- 不同期限的命中率
- 重要性评分有效性
- 漏报、误报与噪音统计
- 研究错误归因：事实、逻辑、时点、期限或执行
