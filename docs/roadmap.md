# 第一版实施路线图

## Phase 0：确定报告成品

目标：先明确每天和每周最终要读什么，再决定收集什么信息。

交付物：

- 日报模板；
- 周报模板；
- 单条信息写作格式；
- 报告长度与数量约束；
- 引用和发布前检查规则。

验收标准：仅靠人工资料即可完成一份结构清晰、可在10至15分钟内读完的日报。

## Phase 1：建立核心信息源库

目标：建立规模有限、质量可控、用途明确的信息源清单。

工作内容：

- 按宏观、美股、A股与港股、贵金属、加密、能源、AI与前沿科技分类；
- 区分官方来源、专业媒体、数据平台、机构研究、Newsletter和社交媒体；
- 记录获取方式、语言、更新频率和报告用途；
- 明确每日、每周和按需查看的来源；
- 标记核心、补充和试验来源；
- 删除重复、低质量和长期无有效信息的来源。

首批控制在20至40个核心来源。

验收标准：每个市场都有少量稳定的一手来源和补充来源，并能说明每个来源为什么值得保留。

## Phase 2：建立每日收集箱

目标：让所有候选信息使用统一格式保存和筛选。

工作内容：

1. 将候选链接和资料放入每日收集箱；
2. 使用AI完成摘要、翻译和基础分类；
3. 人工标记高、中、低重要性；
4. 标记为待选、采用、忽略或已发布；
5. 保留来源、发布时间和原文链接；
6. 记录必要的个人备注。

验收标准：任何进入报告的信息都能回到原始来源，且同一内容不会因格式不统一而重复整理。

## Phase 3：连续人工试运行

目标：在不依赖复杂代码的情况下，跑通日报和周报。

日常流程：

1. 收集候选信息；
2. 去除明显重复和低价值内容；
3. 选择3至5条核心信息；
4. 补充分市场信息、市场数据和事件日历；
5. 按模板生成并人工修改日报；
6. 周末从本周日报和收集箱整理周报；
7. 记录耗时、遗漏、重复和无价值栏目。

试运行周期：至少两个完整交易周。

验收标准：

- 日报和周报能够稳定生成；
- 每条重要内容可回溯来源；
- 每日候选信息处于可人工处理范围；
- 模板经过真实阅读后至少修改一次；
- 可以明确指出最耗时和最容易遗漏的步骤。

## Phase 4：第一版定稿

目标：根据试运行结果，确定第一版正式规则。

需要定稿：

- 保留和删除的信息源；
- 各市场的核心来源数量；
- 日报和周报最终结构；
- 单条信息长度；
- 高、中、低重要性标准；
- 每日整理时间上限；
- 归档和命名方式；
- 第一批确实需要优化的问题。

验收标准：第一版流程可以被重复执行，不依赖临时记忆和随意调整。

## 当前执行顺序

1. 定稿日报和周报格式；
2. 建立首批核心信息源清单；
3. 使用每日收集箱；
4. 连续人工运行两个完整交易周；
5. 根据实际阅读体验调整来源和模板；
6. 完成第一版复盘与定稿。

第一版以外的自动化、内容增强、研究模型和平台建设，统一参考[第二版优化指南](v2-optimization-guide.md)。
