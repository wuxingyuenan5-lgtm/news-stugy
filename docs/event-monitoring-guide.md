# 重要事件监控建议指南

> 文档性质：推荐性指南，不是强制入选清单  
> 当前版本：0.1  
> 适用范围：跨资产早报、晚报与周报

## 1. 文档目的

本指南用于持续维护系统建议关注的重要事件类型，帮助事件采集、筛选、聚类和报告生成模块理解：

- 哪些事件通常具有较高的跨资产研究价值；
- 哪些事件适合持续监控，但不应机械进入报告；
- 哪些事件默认不主动监控；
- 哪些情况下可以突破默认规则，作为重大例外处理；
- 后续新增、删除或调整事件类别时，应依据什么标准。

本指南不直接决定某条事件是否进入报告。最终入选仍取决于事件的实质变化、信息增量、来源可靠性、市场影响、跨资产关联度和报告窗口。

## 2. 非强制性原则

### 2.1 推荐不等于强制入选

即使某类事件被列为“高推荐”，如果当前只有重复报道、模糊传闻或没有实质进展，也不应进入报告正文。

### 2.2 未列出不等于禁止

清单之外的事件如果产生明显市场影响、系统性风险或重要研究价值，系统仍可将其作为重大例外事件处理。

### 2.3 不按类别凑数

报告不要求每个事件类别都出现内容，也不要求每个市场板块平均分配篇幅。

### 2.4 事件数量不代表重要性

大量转载只能提高传播度或确认度，不能机械提高事件的重要性。应区分总来源数量与独立来源数量。

### 2.5 优先确认事实，再解释影响

政策、监管和官方事件优先使用原始文件、监管机构、政府部门、交易所或公司公告确认；专业媒体用于补充事实和背景；研究者与博主主要用于解释影响与呈现分歧。

## 3. 推荐级别

| 级别 | 含义 | 默认处理 |
|---|---|---|
| 高 | 通常具有较高跨资产影响，建议持续监控 | 出现实质进展时优先评估 |
| 中 | 可能影响单一市场或当前主题 | 有明显增量或市场影响时纳入 |
| 低 | 通常噪音较高或影响较窄 | 默认不进入正文，重大例外时处理 |
| 不建议 | 第一版不建议主动监控 | 仅保留重大例外通道 |

推荐级别只影响候选优先级，不构成强制报告规则。

## 4. 通用纳入判断

事件进入报告候选池前，建议至少检查以下因素：

1. **实质变化**：是否从传闻、草案、咨询、审议推进至正式通过、生效或执法；
2. **信息增量**：是否出现新的数字、范围、时间表、执行细则或市场反应；
3. **来源可靠性**：是否存在官方来源、一手媒体或多个独立来源确认；
4. **影响范围**：是否影响主要资产、行业、政策预期或多个市场；
5. **当前相关性**：是否与当前市场主线或用户重点研究方向有关；
6. **可解释性**：是否能够区分已确认事实、合理解释与个人观点；
7. **报告增量**：此前报告是否已经覆盖，当前是否存在值得再次报告的更新。

## 5. 事件状态与更新阶段

政策和监管事件往往持续数日或数月。系统应跟踪状态变化，而不是每天重复介绍背景。

建议使用以下状态：

```text
rumor        传闻或未经确认信息
proposal     提案、草案或初步方案
consultation 公开咨询或征求意见
review       审议、听证或审批中
approved     已获批准但尚未正式生效
enacted      已签署或正式通过
effective    已正式生效
implementation 实施细则或执行安排发布
enforcement  执法、处罚或司法执行
reversed     撤回、推翻、延期或发生方向性反转
```

只有状态变化、关键细节变化、重大市场反应或此前判断被证伪时，才建议再次进入日报。

## 6. 高推荐事件

### 6.1 央行、流动性与财政政策

**默认监控：** 是  
**适用报告：** 早报、晚报、周报

建议关注：

- 主要央行利率决议、资产负债表政策和流动性工具；
- 重要央行官员对政策路径的明确修正；
- 财政刺激、政府支出、财政账户和流动性变化；
- 债务上限、政府停摆、重大国债发行安排；
- 汇率、资本流动及跨境资金政策；
- 影响金融条件的银行和非银监管变化。

通常不纳入：

- 没有新增立场的重复讲话；
- 纯粹复述市场已充分定价的政策预期；
- 无法确认来源的“官员据称”类传闻。

### 6.2 核心宏观数据与重大数据修正

**默认监控：** 是  
**适用报告：** 早报、晚报、周报

建议关注：

- 通胀、就业、经济增长、消费和制造业数据；
- 中国信贷、社融、房地产、工业与消费数据；
- 明显偏离预期的数据；
- 对历史数据的重大修正；
- 改变央行政策路径或经济周期判断的数据组合。

单个数据是否进入正文，应结合预期差、市场反应和当前宏观主线判断。

### 6.3 金融稳定与系统性风险

**默认监控：** 是  
**适用报告：** 早报、晚报、周报

建议关注：

- 银行、券商、清算机构或大型非银机构风险；
- 流动性冻结、挤兑、重大违约或信用事件；
- 主权信用评级和偿债风险；
- 主要交易所、托管机构或稳定币的系统性事故；
- 可能引发跨市场传染的杠杆与保证金事件。

### 6.4 关税、贸易限制与制裁

**默认监控：** 是  
**适用报告：** 早报、晚报、周报

建议关注：

- 影响铜、锂、黄金、能源、半导体和关键设备的关税；
- 出口管制、进口禁令、配额和战略物资政策；
- 重大制裁与反制措施；
- 影响全球供应链、航运和区域价差的政策变化。

### 6.5 重大地缘政治与物流中断

**默认监控：** 是  
**适用报告：** 早报、晚报、周报

建议关注：

- 军事冲突明显升级或范围扩大；
- 重要航道、港口、能源设施或关键物流中断；
- 对能源、贵金属、航运和风险偏好造成明显影响的突发事件；
- 重大停火、谈判或政策反转。

普通政治表态、缺少后续行动的口头威胁和无法核实的战场传闻不应自动进入报告。

## 7. 加密政策与监管

### 7.1 加密立法与监管框架

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 美国、欧盟、香港、中国内地及其他主要司法辖区的加密立法；
- 证券与商品属性认定；
- 加密市场结构、发行、披露和中介机构规则；
- 对DeFi、质押、借贷和衍生品的系统性监管变化。

### 7.2 稳定币监管

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 发行资质和准入门槛；
- 储备资产、审计和赎回要求；
- 银行与非银机构发行权限；
- 跨境支付、反洗钱和制裁要求；
- 对美债需求、加密流动性和主要稳定币产生实质影响的规则。

### 7.3 加密ETF与机构产品

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 主要加密ETF的申请、延期、批准、拒绝和规则变化；
- 实物申赎、质押、期权、托管和指数规则；
- 可能显著改变机构资金流入路径的产品创新；
- 大型养老金、银行、券商和资管机构的制度性准入变化。

### 7.4 交易所、托管与牌照

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 主要交易所和托管机构的牌照、限制经营与退出市场；
- 客户资产隔离、储备证明、KYC和反洗钱规则；
- 杠杆、衍生品和零售投资者限制；
- 对市场流动性和用户资产安全产生实质影响的事件。

普通地区性营销许可、小型平台牌照和不影响主要市场的日常合规更新通常不进入正文。

### 7.5 执法、诉讼与司法裁决

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 主要监管机构对大型交易所、稳定币、项目方和服务商的执法；
- 对证券属性、监管权限和行业规则具有先例意义的裁决；
- 重大和解、处罚、撤诉和上诉结果；
- 制裁、黑名单和大规模资产冻结。

应持续跟踪案件阶段，避免将立案、指控、裁决和最终执行混为一谈。

### 7.6 银行准入、税务与会计规则

**推荐级别：** 中  
**默认监控：** 是

建议关注：

- 银行开展加密托管、结算和交易业务的制度变化；
- 企业持有数字资产的会计处理；
- 对机构持仓、企业财库和投资者行为影响明显的税务变化；
- 资本占用、风险权重和金融机构审慎监管。

## 8. 商品、能源与供应链

### 8.1 重大供给中断

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 重要铜矿、锂矿、金矿、油气田的停产、复产、罢工和事故；
- 对全球供给平衡具有可量化影响的产量调整；
- 战略储备收储和抛储；
- OPEC及主要产油国的产量政策。

普通项目进度、未经确认的小型矿山传闻和没有产量影响估算的信息通常只归档。

### 8.2 产业政策与产能变化

**推荐级别：** 中  
**默认监控：** 是

建议关注：

- 铜、锂、贵金属、能源和电池产业的重要产能投放；
- 对供需平衡产生实质影响的审批、环保和资源政策；
- 冶炼费、加工费、库存和区域溢价出现结构性变化；
- 交易所交割、保证金和仓储规则的重大调整。

## 9. AI与半导体

### 9.1 芯片与关键设备政策

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- AI芯片、半导体设备和关键材料的出口限制；
- 影响GPU、存储、先进制程和供应链的政策；
- 主要国家的产业补贴、投资限制和技术管制。

### 9.2 AI资本开支与基础设施

**推荐级别：** 高  
**默认监控：** 是

建议关注：

- 大型云厂商和AI公司的资本开支与指引；
- GPU、存储、数据中心、电力和网络设备瓶颈；
- 大型融资、采购、供应协议和项目延期；
- 能够改变AI产业链利润分配或市场预期的事件。

### 9.3 重大模型与产品发布

**推荐级别：** 中  
**默认监控：** 是

只有在以下情况下建议进入跨资产报告：

- 显著改变模型能力、成本或商业化路径；
- 直接影响大型上市公司、半导体需求或云计算竞争；
- 引发明确的市场定价或监管反应。

普通功能升级、营销发布和缺乏可验证增量的模型宣称通常不进入正文。

### 9.4 AI监管、版权与数据政策

**推荐级别：** 中  
**默认监控：** 是

建议关注对模型训练、数据获取、版权责任、企业部署和跨境技术服务具有实质影响的法规、裁决和行业规则。

## 10. 公司与产业事件

### 10.1 主要科技与跨资产关键公司的财报和指引

**推荐级别：** 高  
**默认监控：** 是

重点关注：

- 大型云厂商、半导体、AI基础设施和关键供应链公司；
- 影响资本开支、算力需求、存储价格和行业景气的指引；
- 具有明显指数或行业外溢影响的业绩变化。

### 10.2 普通公司新闻

**推荐级别：** 低  
**默认监控：** 否

只有在影响主要指数、行业供需、系统性风险或当前核心主题时，才建议作为重大例外纳入。

## 11. 第一版默认不建议主动监控

以下事件第一版默认关闭常规主动监控：

- 普通代币解锁；
- 常规区块链硬分叉；
- 普通协议升级；
- 空投、积分活动和项目营销活动；
- 普通交易所上币、下币；
- 小型项目治理投票；
- 一般性主网路线图更新；
- 单一小项目融资和合作公告；
- 缺少跨资产影响的短期技术分析信号。

默认不监控并不表示永久删除。未来如果某类事件连续产生研究价值，可以通过修改本指南和运行配置重新启用。

## 12. 重大例外机制

默认不建议监控的事件，如果满足以下任一条件，仍可作为重大例外进入候选池：

- 影响BTC、ETH或主要稳定币的安全、供给或市场结构；
- 导致主要交易所、协议或托管机构发生系统性风险；
- 引发显著价格波动、流动性变化或跨市场传染；
- 改变ETF、监管、机构准入或关键会计处理；
- 获得官方机构或多个高质量独立来源确认；
- 与当前报告核心主线高度相关，且提供明显新增信息。

配置层建议保留：

```yaml
default_monitoring: false
allow_material_exception: true
```

## 13. 事件来源建议

建议按以下顺序确认事件：

1. 官方政府、央行、监管机构、交易所和公司公告；
2. 官方数据与正式法律、监管文件；
3. 一手专业媒体和专业数据机构；
4. 垂直媒体与高质量研究机构；
5. 研究者、公众号、YouTube和X账号；
6. 普通转载、快讯和未验证传闻。

社交媒体可以用于发现线索，但重要事实原则上应寻找更高等级来源确认。

## 14. 报告使用方式

### 14.1 已发生事件

进入核心事件或市场主线时，建议写明：

- 当前阶段；
- 本次新增信息；
- 与此前报道相比的变化；
- 可能影响的资产；
- 已确认事实与尚未明确事项；
- 主要来源和独立来源数量。

### 14.2 预定事件

明确时间的会议、投票、数据和政策节点，可进入“今日关注”“今晚及明晨关注”或“下周关注”。

### 14.3 轻微更新

只有重复报道、措辞变化或非实质细节时，只更新事件状态和数据库，不重复进入报告。

### 14.4 分歧处理

不同来源对事件影响存在明显分歧时，应在同一主题下并列展示，不应平均化或强行形成单一结论。

## 15. 维护与优化方法

建议每两至四周根据实际运行数据复盘一次：

- 某类事件被采集的数量；
- 最终进入报告的比例；
- 重复率和低价值内容比例；
- 产生重大遗漏的事件类型；
- 来源完整性和抓取失败率；
- 是否需要新增、降级、升级或停用某一类别。

调整流程建议为：

```text
提出调整理由
→ 修改本指南
→ 记录变更原因
→ 再修改运行配置
→ 观察两至四周
→ 复盘效果
```

本指南与运行配置应保持解耦：

- `docs/event-monitoring-guide.md`记录推荐逻辑与判断依据；
- `config/event-monitoring.yaml`记录程序当前实际执行的类别和优先级；
- 修改文档不会自动改变线上运行配置。
