# 零付费内容与数据策略 V1

> 文档性质：第一版硬约束与实施建议  
> 当前版本：0.1  
> 适用范围：新闻、社交媒体、公众号、YouTube、行情、宏观数据、宏观日历和政策事件来源

## 1. 约束边界

第一版不为内容与数据来源付费，包括：

- 付费新闻订阅和付费墙正文；
- 按调用量计费或要求预充值的内容API；
- 商业行情、宏观日历和一致预期接口；
- X官方付费API；
- 付费代理、住宅代理、验证码代解和商业网页解析服务；
- 付费字幕、音频下载、转写或媒体监控服务。

本约束不限制AI模型与Token使用。摘要、翻译、分类、聚类、评分和报告生成可以调用外部AI模型，也可以使用本地模型；模型选择由质量、成本和稳定性另行决定。

## 2. 核心原则

1. **免费优先，不伪装稳定**：免费来源可以作为正式方案，但必须记录其非官方性、限流和失效风险。
2. **官方免费来源优先**：经济数据、监管文件、国债收益率、交易所公告等优先使用官方公开接口或下载文件。
3. **多Provider回退**：X、公众号、行情等不稳定来源必须允许切换Provider。
4. **缺失允许，编造禁止**：免费来源无法取得某项数据时，报告标记缺失，不使用来源不明的数据补齐表格。
5. **原始内容可追溯**：每条事实应保留原始链接、获取时间、Provider和内容完整度。
6. **个人研究用途**：使用公开网页或非官方工具时，应遵守相应网站条款，不将数据重新包装为商业数据服务。

## 3. 新闻与专业媒体

### 3.1 获取顺序

```text
官方RSS或公开Feed
→ 官网公开栏目和文章页
→ 搜索发现与公开摘要
→ 其他免费只读镜像或解析方式
```

### 3.2 处理规则

- 公开正文可正常提取、摘要和引用；
- 付费墙内容只使用合法可见的标题、摘要、导语和元数据；
- 不根据标题猜测付费正文；
- 同一事实优先回溯官方文件或可访问的一手来源；
- Bloomberg、Reuters、财新、Fastmarkets等即使无法稳定取得全文，仍可作为事件发现和来源引用，但报告不得声称已读取不可访问正文。

## 4. X

X官方API不作为第一版方案，因为内容读取需要付费额度。

### 4.1 免费Provider顺序

```text
参考项目现有X采集链
→ 可用的Nitter或兼容实例
→ 公开搜索结果和网页解析
→ 作者其他公开渠道交叉补充
```

### 4.2 运行要求

- 每个Provider独立封装；
- 使用作者、帖子ID、URL、时间和内容哈希去重；
- 不抓普通回复，保留作者自己的连续线程；
- 原创、引用和转发分别保存；
- 纯转发通常只作为线索或热度信号；
- 时间不精确时标记`approximate`；
- 无法取得正文时记录失败，不根据搜索摘要扩写原帖观点。

### 4.3 稳定性定位

X属于第一版最不稳定的免费来源之一。系统必须允许部分账号在某个报告窗口缺失，并在运行日志中记录覆盖率。

## 5. 公众号

公众号继续以参考项目已经验证的文章发现逻辑为主：

```text
公众号配置或种子文章
→ 免费Provider发现近期文章
→ 微信文章页解析正文
→ 图片和长图补充
```

要求：

- Provider与正文解析器分离；
- 只处理报告窗口内发布的新文章；
- 公开文章正文可正常摘要；
- 无法访问的文章不根据标题推断；
- 第三方免费接口失效时只影响公众号模块，不阻塞整份报告；
- 连续失败时自动降频并记录失败率。

## 6. YouTube

### 6.1 视频发现

优先使用YouTube Data API的免费配额，通过频道上传播放列表发现新视频；yt-dlp作为免费备用发现方式。

### 6.2 内容获取

```text
youtube-transcript-api
→ yt-dlp人工字幕
→ yt-dlp自动字幕
→ yt-dlp下载音频
→ 本地或外部AI转写
```

AI转写服务本身不受“内容零付费”约束，但不得购买第三方字幕或媒体监控数据服务。

### 6.3 过滤规则

- 排除直播、直播回放和Shorts；
- 不能仅按时长判断Shorts；
- 保留字幕来源、语言、完整度和时间戳；
- 长视频分段摘要后再合并；
- 字幕、音频和有效画面都不可得时不生成内容摘要。

## 7. 行情数据

### 7.1 海外权益、ETF、外汇和商品

第一版采用免费公开工具与网页接口：

- 主Provider：`yfinance`；
- 备用Provider：Stooq或其他公开日线来源；
- 报告只需要固定时点或区间变化，不追求专业终端级实时行情；
- 对来源异常值、拆分和缺失数据进行校验。

覆盖：

- 标普500、纳斯达克100；
- SMH；
- 美元指数、USD/CNH；
- 黄金、白银、原油；
- COMEX铜。

### 7.2 中国期货

- 沪铜：AkShare或上期所公开页面；
- 碳酸锂：AkShare或广期所公开页面；
- 优先日行情和主力合约口径；
- 网页接口变化时允许暂时缺失。

### 7.3 加密资产

BTC、ETH使用Binance公开市场数据接口；必要时预留OKX或Bybit公开行情作为备用。

### 7.4 债券收益率

- 美债2Y、10Y、30Y：美国财政部官方日度收益率；
- 中债2Y、10Y、30Y：中债收益率曲线公开日终数据；
- 日度数据不得包装成盘中实时变化；
- 中债当日数据未发布时使用最近可用交易日并注明日期。

## 8. 宏观数据与日历

### 8.1 实际值与历史值

优先使用：

- FRED；
- BLS；
- BEA；
- 美联储、美国财政部；
- 中国国家统计局、人民银行、海关总署；
- 其他央行和统计机构公开数据。

### 8.2 宏观日历

第一版不购买Trading Economics等商业日历API，采用：

```text
金十公开财经日历
+ 官方数据发布日程
+ 央行会议和讲话日历
+ 财政部拍卖日历
+ 公司财报公开日历
```

### 8.3 一致预期

一致预期是免费方案的主要缺口。处理规则：

- 金十等公开页面能够取得预期值时保存；
- 无法稳定取得时只展示前值、实际值和官方说明；
- 不从单一博客或来源不明页面补齐预期值；
- 没有预期值时，不计算“超预期/低于预期”；
- 可以改为描述“相较前值变化”和公布后的市场反应。

## 9. 免费来源降级等级

```yaml
content_status:
  complete: 已取得完整正文、字幕或数据
  partial: 只取得部分正文、摘要或部分字段
  metadata_only: 只有标题、时间和链接
  unavailable: 本次获取失败
```

报告正文使用规则：

- `complete`：可正常进入分析；
- `partial`：只总结已取得部分并标明范围；
- `metadata_only`：通常只作为事件线索，不生成详细观点；
- `unavailable`：仅记录日志和完整度，不进入报告。

## 10. 禁止自动切换到付费服务

系统不得因免费Provider失败而自动调用付费内容服务。任何未来新增的付费内容源都必须由用户明确修改本约束后启用。

配置建议：

```yaml
content_data_cost_policy:
  mode: zero_paid_sources
  allow_paid_news: false
  allow_paid_market_data: false
  allow_paid_macro_calendar: false
  allow_paid_social_api: false
  allow_paid_proxy: false
  ai_model_cost_restricted: false
```

## 11. 验收指标

除报告质量外，应持续记录：

- 各平台采集成功率；
- 完整正文或字幕取得率；
- 元数据-only比例；
- 免费Provider失效率；
- 行情字段缺失率；
- 宏观预期值覆盖率；
- 因免费来源限制产生的漏报案例；
- Provider切换后的改善或退化。

## 12. 后续优化

每两至四周复盘一次。优先优化顺序：

1. 提高官方免费来源覆盖；
2. 改善参考项目采集逻辑；
3. 增加同类免费备用Provider；
4. 降低对单个网页结构的依赖；
5. 删除长期低成功率且低价值的来源；
6. 在不改变零付费内容约束的前提下调整标的代表口径。
