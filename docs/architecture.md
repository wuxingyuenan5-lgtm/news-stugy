# 跨资产信息整理系统架构 V1

## 1. 第一版定位

第一版只建立一套稳定、可持续的个人信息整理与报告生产流程，优先回答：

1. 每天应该看哪些来源；
2. 哪些信息值得进入日报或周报；
3. 信息如何分类、压缩和呈现；
4. 如何保证内容可回溯、不重复、不过度堆积。

所有不属于这一最小闭环的能力，统一收录在[第二版优化指南](v2-optimization-guide.md)中。

## 2. 第一版工作流

```text
信息源登记
  ↓
候选信息收集
  ↓
基础清洗与去重
  ↓
市场 / 内容类型分类
  ↓
人工筛选与编辑
  ↓
日报 / 周报生成
  ↓
历史归档
```

AI主要负责：

- 摘要；
- 翻译；
- 基础分类；
- 明显重复提示；
- 按模板整理。

人工主要负责：

- 决定信息是否重要；
- 修改标题和摘要；
- 补充必要背景；
- 确认最终报告内容与结构。

## 3. 第一版核心模块

### 3.1 信息源管理

维护一份规模有限、用途明确的信息源清单。每个来源只记录：

- 名称；
- 链接或获取方式；
- 覆盖市场；
- 内容类型；
- 语言；
- 更新与查看频率；
- 来源等级；
- 核心、补充或试验优先级；
- 是否启用；
- 在报告中的主要用途；
- 备注。

来源等级：

- A级：官方机构、交易所、公司公告和原始数据；
- B级：主流财经媒体、专业数据平台和高质量研究机构；
- C级：Newsletter、社交媒体、博主和其他线索来源。

等级表示事实可靠性和原始程度，不代表观点必然正确。

### 3.2 每日信息收集箱

当天候选信息统一进入收集箱。第一版可使用Markdown、YAML、JSON或CSV，不要求数据库。

```yaml
id:
published_at:
source:
title:
url:
markets: []
content_type:
summary:
importance: high | medium | low
status: inbox | selected | ignored | published
analyst_note:
```

其中：

- `summary`只写事实摘要；
- `importance`只使用高、中、低三级；
- `analyst_note`为可选备注；
- 每条内容必须保留原始链接。

### 3.3 人工筛选与编辑

优先进入报告的信息通常满足至少一项：

- 重大政策、经济数据、公司公告或产业变化；
- 相比此前出现明确新增信息；
- 与当前重点跟踪市场或主题直接相关；
- 能帮助理解重要资产的显著变化；
- 未来数日仍值得继续跟踪。

通常不进入报告：

- 只有标题、没有可靠事实；
- 多家媒体重复转述同一内容；
- 与覆盖范围关系较弱的泛泛评论；
- 连续多日出现但没有新进展；
- 仅围绕短期价格波动的事后归因。

### 3.4 报告生成与归档

日报和周报使用固定Markdown模板。每条重要内容至少包含：

- 标题；
- 事实摘要；
- 来源和链接；
- 所属市场；
- 可选的一句简要意义。

报告完成后按日期归档，便于后续检索和模板调整。

## 5. 第一版分类体系

市场分类：

- macro；
- us_equity；
- china_equity；
- precious_metals；
- crypto；
- energy；
- ai_frontier_tech。

内容类型：

- data：经济与市场数据；
- policy：政策、监管与央行；
- company：公司财报与公告；
- industry：产业链与供需；
- market：市场表现、资金流与仓位；
- research：机构和个人研究观点；
- calendar：未来事件。

主题标签保持克制，只有被持续使用后才加入正式字典。

## 6. 第一版验收标准

连续试运行两个完整交易周，并达到：

- 建立20至40个核心信息源；
- 每日候选信息处于可人工阅读范围；
- 日报和周报能够稳定产出；
- 每条重要内容均可回溯原始来源；
- 重复和低价值内容明显减少；
- 模板经过真实阅读后至少调整一次；
- 能记录整理耗时、漏报和无价值栏目。

第一版成功的标志不是功能多，而是愿意持续使用。
