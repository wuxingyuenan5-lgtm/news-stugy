# 日报与周报规范 V0.1

## 1. 输出原则

报告不是新闻列表。每个核心条目至少包含：

- 新发生的事实
- 市场原预期或既有叙事
- 预期差
- 跨资产传导路径
- 影响方向与期限
- 后续验证指标

事实、推断和观点必须明确区分。

## 2. 日报

### 2.1 目标

识别最近一个报告周期内的增量变化，并说明哪些变化值得继续跟踪。

### 2.2 固定结构

```markdown
# 跨资产日报｜YYYY-MM-DD

## 一、核心结论
- 3–5 条结论，每条尽量不超过两句话。

## 二、跨资产市场快照
- 美债、美元、美股、A股、黄金、原油、BTC 等核心资产表现。
- 只描述显著变化，并给出主要归因。

## 三、关键事件
### 1. 事件标题
**事实：**
**预期差：**
**传导路径：**
**资产影响：**
**观察指标：**
**置信度：**

## 四、快速信息
- 5 条以内，每条包含一句事实和一句意义。

## 五、资金流、仓位与波动率

## 六、今日事件日历

## 七、待验证问题
- 2–4 个未来需要数据或价格行为确认的问题。
```

### 2.3 数量约束

- 核心结论：3–5 条
- 关键事件：最多 5 条
- 快速信息：最多 5 条
- 待验证问题：2–4 条

信息不足时宁可少写，不为填满版面加入噪音。

## 3. 周报

### 3.1 目标

判断本周变化是否构成趋势、叙事强化、叙事反转或市场状态切换。

### 3.2 固定结构

```markdown
# 跨资产周报｜YYYY-Www

## 一、本周核心判断

## 二、市场状态
- 增长
- 通胀
- 流动性
- 货币政策
- 风险偏好
- 资金流与仓位

## 三、资产表现与归因
- 美股
- A股与港股
- 贵金属
- 加密
- 能源
- AI与前沿科技

## 四、本周关键事件与传导

## 五、当前核心叙事
- 强化中的叙事
- 弱化中的叙事
- 新出现的非共识线索

## 六、观点验证
- 得到验证
- 被削弱
- 已失效
- 尚未验证

## 七、下周风险日历

## 八、情景推演
- 基准情景
- 乐观情景
- 悲观情景

## 九、重点观察资产与触发条件
```

## 4. 写作规则

1. 先写结论，再写证据。
2. 避免把价格变化本身当作原因。
3. 不使用“市场认为”而不说明证据来源。
4. 方向判断必须附带期限。
5. 条件性影响必须明确条件。
6. AI 无法确认的事实不得写成确定语气。
7. 同一事件更新时只写新增部分，并链接原事件。

## 5. 引用规范

每个关键事件保留：

- 主来源
- 佐证来源
- 原始发布时间
- 系统抓取时间
- 原文或本地存档位置

最终 Markdown 报告可以使用脚注或文末引用表，但内部数据库必须保留文档 ID。

## 6. 审核清单

发布前检查：

- 是否存在无来源事实
- 是否混淆事件发生时间与报道时间
- 是否遗漏市场预期
- 是否把相关性写成因果关系
- 是否给出影响期限
- 是否有相反情景或失效条件
- 是否重复昨天已知内容
- 是否存在AI生成但无法回溯原文的数字
