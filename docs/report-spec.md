# 早报、晚报与周报规范 V1

## 1. 输出目标

第一版固定生成三类Markdown报告：

- 早报：每日北京时间08:00；
- 晚报：每日北京时间18:00；
- 周报：每周六北京时间18:00。

报告用于：

- 提炼报告窗口内真正有增量的信息；
- 将新闻、数据、政策、市场价格和研究观点放入统一框架；
- 识别同一事件的更新、补充、市场反应和分歧；
- 明确区分事实、解释、观点与预测；
- 保留原始来源，便于回溯；
- 为下一交易时段或下一周提供事件日历和验证点。

报告不按平台平均分配内容，也不为填满栏目加入低价值信息。

## 2. 报告窗口

### 2.1 早报

- 内容窗口：前一日18:00至当日08:00；
- 生成时间：当日08:00。

### 2.2 晚报

- 内容窗口：当日08:00至18:00；
- 生成时间：当日18:00。

### 2.3 周报

- 内容窗口：上周六18:00至本周六18:00；
- 生成时间：本周六18:00；
- 不拼接日报，按主题和事件演变重新综合。

## 3. 写作总原则

### 3.1 事实与观点分开

核心条目应尽量区分：

```text
事实：已经发生且可核实的内容
解释：为什么重要以及可能影响什么
观点：媒体、研究者或市场参与者的判断
未知：尚未确认的范围、时间表或后续结果
```

### 3.2 不强行归因

事件影响使用三级置信度：

- `confirmed`：数据、时间反应和可靠来源均支持；
- `likely`：可能是主要驱动之一，但无法排除其他因素；
- `unclear`：市场反应有限、混杂或暂时无法解释。

找不到单一驱动时，应明确写“暂未发现单一明确驱动”。

### 3.3 信息簇而非单条来源

同一事件只形成一个报告条目：

- 转载和重复不重复展示；
- 新增数字、政策细节、产业影响和市场反应合并写入；
- 相似观点归纳共同结论；
- 相反观点并列展示；
- 每个事件原则上最多显示三个代表性来源。

### 3.4 跨报告去重

早报已写过的内容，晚报只有在出现实质更新、方向反转或明确市场确认时再次进入，并标记为“更新”。周报重新总结一周内的演变。

## 4. 单个核心条目格式

普通核心事件建议控制在150至300字；重大政策、关键数据或跨资产事件可扩展至300至500字。

```markdown
### 事件标题

**事实：** 说明发生了什么，以及相较此前新增了什么。

**影响：** 说明对市场预期、政策路径、产业供需或资产定价的影响。无法确认时使用谨慎表述。

**分歧或未知：** 可选。列出尚未明确事项或主要相反观点。

**涉及资产：** 美股、A股/港股、美债、黄金、铜、BTC等实际相关资产。

**信息性质：** 事实 / 数据 / 分析 / 观点 / 预测

**来源：** 来源名称｜发布时间｜原文链接
```

不是每条都必须出现全部字段，但事实、来源和影响边界必须清楚。

## 5. 跨资产市场看板

市场看板是早报和晚报固定模块。第一版重点展示：

### 5.1 权益与风险偏好

- 标普500；
- 纳斯达克100；
- 沪深300；
- 恒生指数；
- SMH半导体ETF，作为AI基础设施和半导体代理。

### 5.2 利率与汇率

- 美债2Y、10Y、30Y收益率及BP变化；
- 中债2Y、10Y、30Y收益率及BP变化；
- 2Y—10Y和10Y—30Y曲线变化；
- 美元指数；
- 美元兑人民币。

### 5.3 商品

- 黄金；
- 白银；
- 铜，优先采用稳定可得的COMEX铜或沪铜主力；
- 广期所碳酸锂主力；
- 原油。

### 5.4 加密

- BTC；
- ETH。

市场看板展示最新水平、窗口变化和必要的简要说明。债券展示收益率而不是价格。

```markdown
## 跨资产市场看板

| 类别 | 标的 | 最新水平 | 窗口变化 | 简要说明 |
|---|---|---:|---:|---|
| 美股 | 标普500 |  |  |  |
| 利率 | 美债10Y |  | BP |  |
| 商品 | 黄金 |  |  |  |
| 加密 | BTC |  |  |  |
```

无可靠数据时标记“暂无可靠数据”，不使用来源不明数据补齐表格。

## 6. 宏观数据与事件模块

报告固定区分“已经发生”和“接下来关注”。

### 6.1 已公布数据与已发生事件

```markdown
## 已公布宏观数据与事件

| 北京时间 | 数据/事件 | 前值 | 预期 | 实际 | 结果判断 | 市场影响 |
|---|---|---:|---:|---:|---|---|
```

最重要的数据或事件可以在表格后展开：

- 实际与预期差异；
- 公布后的美债、美元、权益、黄金或BTC反应；
- 影响置信度；
- 是否改变政策或市场预期；
- 尚需验证的分项或后续事件。

### 6.2 后续事件日历

```markdown
## 接下来关注

| 北京时间 | 数据/事件 | 市场预期或当前阶段 | 重点观察 | 主要影响资产 |
|---|---|---|---|---|
```

事件包括：

- 经济数据；
- 央行会议与讲话；
- 美债拍卖；
- 财报；
- 政策与监管节点；
- 商品产业会议与供给节点；
- 重要加密监管、ETF和稳定币事件；
- AI与半导体重要发布或政策。

普通代币解锁、常规硬分叉和普通协议升级默认不列入日历，重大例外按事件监控指南处理。

## 7. 早报结构

目标：说明隔夜发生了什么、市场如何定价，以及今天白天需要关注什么。

```markdown
# 跨资产早报｜YYYY-MM-DD

报告窗口：YYYY-MM-DD 18:00—YYYY-MM-DD 08:00
生成时间：北京时间08:00

## 一、隔夜核心结论
## 二、跨资产市场看板
## 三、隔夜宏观数据与事件复盘
## 四、核心新闻与市场主线
## 五、重点观点与分歧
## 六、今日宏观数据与重要事件日历
## 七、其他值得浏览
## 八、数据完整性说明
```

### 7.1 隔夜核心结论

- 4至6条为常态；
- 每条一句话；
- 不是新闻标题堆叠，而是对市场状态的归纳；
- 信息较少时可以少于4条。

### 7.2 核心新闻与市场主线

- 通常5至10个信息簇；
- 重大数据、政策和跨资产事件优先；
- 同一主题只出现一次。

### 7.3 重点观点与分歧

- 优先按主题合并博主、公众号、YouTube和研究内容；
- 通常2至5个主题；
- 交易观点不能成为报告主体；
- 明确写出共同依据、主要分歧和待验证点。

### 7.4 其他值得浏览

- 5至15条短信息为参考上限；
- 只保留明确有增量但影响较小的内容；
- 每条附原始链接，不展开长分析。

## 8. 晚报结构

目标：复盘亚洲交易时段、更新早报主题，并为欧美时段做准备。

```markdown
# 跨资产晚报｜YYYY-MM-DD

报告窗口：YYYY-MM-DD 08:00—18:00
生成时间：北京时间18:00

## 一、日内核心结论
## 二、跨资产市场看板
## 三、日内宏观数据与事件复盘
## 四、A股、港股与中国资产主线
## 五、核心新闻与市场主线
## 六、重点观点与分歧
## 七、今晚及明晨宏观日历
## 八、其他值得浏览
## 九、数据完整性说明
```

### 8.1 中国资产主线

重点提炼：

- 指数和市场风格；
- 成交与风险偏好；
- 强势和弱势行业；
- 政策与监管；
- 人民币、中债和商品之间的联系；
- 隔夜主题是否被亚洲市场确认或证伪。

不做简单涨跌幅流水账。

### 8.2 相较早报的变化

重复出现的核心事件应补充：

```markdown
**相较早报：** 说明从传闻到确认、从细节到正式方案、从无定价到市场确认等实质变化。
```

无实质变化则不重复写。

## 9. 周报结构

目标：回答本周主线是什么、如何演变、资产如何定价、下周需要验证什么。

```markdown
# 跨资产周报｜YYYY年第WW周

覆盖窗口：YYYY-MM-DD 18:00—YYYY-MM-DD 18:00

## 一、本周核心结论
## 二、本周市场主线
## 三、各市场周度复盘
## 四、高质量观点与深度研究
## 五、本周共识与分歧
## 六、本周关键宏观数据与事件复盘
## 七、下周宏观数据与重要事件日历
## 八、研究与方法
## 九、数据完整性说明
```

### 9.1 本周市场主线

通常选择3至6个主题，每个主题包含：

```markdown
### 主线标题

**本周演变：**
**关键数据与事件：**
**市场定价变化：**
**跨资产影响：**
**当前分歧：**
**下周验证点：**
```

### 9.2 各市场周度复盘

可覆盖：

- 宏观与流动性；
- 美债与美元利率；
- 中债与中国金融条件；
- 美股；
- A股与港股；
- 黄金与贵金属；
- 铜、锂与工业原材料；
- 能源；
- 加密市场；
- AI与科技。

没有重要增量的板块可省略，不平均分配篇幅。

### 9.3 研究与方法

用于收录：

- 量化策略；
- 交易和研究框架；
- AI工具；
- 高质量知识科普；
- 非时效性但具有长期价值的公众号和YouTube内容。

## 10. 篇幅与数量控制

### 10.1 早报和晚报

- 目标阅读时间：8至12分钟；
- 建议篇幅：3000至5000字；
- 核心事件：通常5至10个；
- 重点观点主题：通常2至5个；
- 其他值得浏览：最多15条。

### 10.2 周报

- 目标阅读时间：20至30分钟；
- 建议篇幅：8000至15000字；
- 市场主线：通常3至6个；
- 本周核心结论：通常6至10条。

以上均为上限或建议区间，不设置强制下限。信息少时宁可缩短。

## 11. 来源与引用规范

每个核心信息簇至少保存：

- 主来源；
- 原始标题；
- 发布时间；
- 原始链接；
- 必要的补充来源；
- 总来源数量；
- 独立来源数量。

来源优先顺序：

1. 官方文件和原始数据；
2. 官方机构、交易所和公司公告；
3. 一手专业媒体；
4. 专业数据机构和垂直媒体；
5. 高质量研究者和博主；
6. 普通转载和未验证线索。

## 12. 数据完整性说明

报告末尾可自动展示：

```markdown
## 数据完整性说明

- 专业媒体：成功X/Y
- X账号：成功X/Y
- 公众号：成功X/Y
- YouTube：成功X/Y
- 行情与宏观接口：成功X/Y
- 主要缺失：……
```

单个来源失败不阻塞报告，但不能掩盖影响内容完整性的重大缺失。

## 13. 自动发布前检查

- 所有事实能否回溯到原始或可靠来源；
- 是否将观点、预测或事后解释写成事实；
- 是否存在上一份报告已经覆盖且没有新增的内容；
- 同一事件是否被多个栏目重复展示；
- 是否遗漏重大官方数据、政策或方向性反转；
- 市场驱动是否有足够证据；
- 债券收益率与价格口径是否混淆；
- 宏观数据的前值、预期、实际值和单位是否正确；
- 链接、日期、时区和报告窗口是否正确；
- 是否为填满栏目加入了低价值内容；
- 报告是否符合目标阅读时间。
