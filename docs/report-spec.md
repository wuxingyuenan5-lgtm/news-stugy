# 日报与周报规范 V0.2

## 1. 第一阶段输出目标

第一阶段报告的任务是：

- 帮助个人快速了解过去一个周期内的重要增量信息；
- 将不同市场的信息放在统一格式中；
- 保留原始来源，便于继续阅读；
- 逐步形成稳定的阅读习惯和重点关注清单。

第一阶段不要求每条信息都完成完整的预期差、传导机制和资产影响分析。

## 2. 单条信息的写法

推荐使用以下轻量格式：

```markdown
### 信息标题

**事实摘要：** 1至3句话，说明发生了什么。

**简要意义：** 可选。只在判断较明确时补充一句。

**来源：** 来源名称｜发布时间｜原文链接
```

写作要求：

- 事实摘要和个人判断分开；
- 不确定的内容使用谨慎语气；
- 没有新增信息时不重复写；
- 同一事件存在多篇报道时，优先引用原始来源或质量最高的报道；
- 一条内容原则上控制在100至250字。

## 3. 日报

### 3.1 日报解决的问题

日报回答：过去24小时有哪些信息值得我知道，今天还需要关注什么。

### 3.2 建议结构

```markdown
# 跨资产日报｜YYYY-MM-DD

## 一、今日最重要的3至5件事

每条包含：标题、事实摘要、可选意义、来源。

## 二、分市场信息

### 宏观
### 美股
### A股与港股
### 贵金属
### 加密资产
### 能源
### AI与前沿科技

没有重要增量的栏目可以留空或省略。

## 三、市场数据速览

只展示最值得关注的价格、利率、汇率、波动率或资金流变化。

## 四、今日与未来事件

列出经济数据、央行会议、财报、政策会议和其他重要时间点。

## 五、持续跟踪

保留2至5个尚未得出结论、需要继续观察的问题。
```

### 3.3 数量控制

- 今日最重要：3至5条；
- 各市场补充：合计5至15条；
- 持续跟踪：2至5条；
- 全文优先控制在可于10至15分钟内读完的范围。

信息较少时宁可缩短，不为填满栏目加入噪音。

## 4. 周报

### 4.1 周报解决的问题

周报不是七份日报的拼接，而是回答：本周哪些信息值得保留，哪些主题正在持续发展，下周应该重点看什么。

### 4.2 建议结构

```markdown
# 跨资产周报｜YYYY-Www

## 一、本周核心摘要

用5至8条概括本周最重要变化。

## 二、本周重要主题

按主题整理，而不是机械重复每日新闻。每个主题包含：
- 本周新增信息
- 当前进展
- 尚未解决的问题

## 三、分市场回顾

### 宏观
### 美股
### A股与港股
### 贵金属
### 加密资产
### 能源
### AI与前沿科技

## 四、本周重要数据与图表

只保留能够帮助理解市场的关键数据。

## 五、下周关注事项

列出数据、会议、财报、政策节点和持续观察主题。

## 六、个人研究备注

记录本周值得进一步研究的线索、疑问和资料。
```

## 5. 信息排序规则

报告内的信息优先按以下顺序排列：

1. 重要政策、经济数据、公司公告和原始事实；
2. 对多个市场有明显影响的重要变化；
3. 当前重点跟踪主题的新增进展；
4. 有价值的机构研究和非共识观点；
5. 一般市场评论和背景资料。

不使用复杂分数。编辑者只需要将信息标记为：

- `high`：应进入报告重点位置；
- `medium`：可作为补充；
- `low`：归档或忽略。

## 6. 引用规范

每条重要信息至少保存：

- 来源名称；
- 原始标题；
- 发布时间；
- 原文链接；
- 必要时保存第二来源。

最终报告应能够从摘要直接回到原文。

## 7. 发布前检查

- 是否有无法回溯来源的事实；
- 是否将评论写成事实；
- 是否重复昨日已知内容；
- 是否遗漏明显重要的信息；
- 是否存在过长摘要；
- 是否为了完整而保留低价值内容；
- 标题、日期和链接是否正确；
- 报告是否能在合理时间内读完。

## 8. 后续增强项

运行稳定后，可以逐步增加：

- 市场预期与实际值；
- 简要影响分析；
- 事件更新链；
- 主题进展跟踪；
- 自动图表；
- 历史搜索与观点复盘。

这些字段应根据实际使用需求增加，而不是一次性全部设计。
